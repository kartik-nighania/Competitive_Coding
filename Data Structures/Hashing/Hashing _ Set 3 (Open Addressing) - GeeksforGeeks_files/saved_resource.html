                  I believe the post should clearly mention the difference between 2 types of assumptions
                  simple uniform hashing and uniform hashing

                  simple uniform hashing - used in chaining - it states that every index has an equal probability of getting selected after computing hash(X). it is independent of all past data.

                  uniform hashing - used in open addressing - It states that every hash function will probe to all the node ie probing sequence of any hash(X) will we a permutation of all possible indexes.

                                SHREYAS SINGH Shashi Pandey • 2 months ago

                                I will rather explain it with an example.
                                Lets say you have to insert X0 and hash(X0,1)=0 , so into index 0;
                                X1 into index 1;
                                X2 into index 2;
                                Now you try to insert X3 and on 1st attempt you get 0 , now since index is already occupied so you move on till you get an empty slot , lets say index 4;
                                Now you delete X0 ie replace data at index 0 with null.(if you dont use delete flag)

                                Now you search for X3 , you first look at hash(X3) which gives index 0. Now since its empty so you return that key dosent exist .(Which is WRONG).

                                But if you would use a delete_me flag , than you can be sure that a key once existed here that is now deleted , so you can keep probing till you find the key or have tested all the indexes.

                                This way we can overcome the problem described above. Hope this helps :)

                                So in summary - insert treats delete_me flag as null while search treats delete_me flag as a key that does not match with the key we are searching for.

                                    Aviral Verma Shashi Pandey • 4 months ago

                                    Hash Table when has collisions and they are resolved by open addressing, then in layman terms, the collision is resolved by putting the key into the next empty slot.

                                    When a search function works, it tries to find the key either by hash function(where the value should be ), if it's not there that means there is collision and search moves to the next empty index! step by step.So if finds a empty slot this means that the key is not present there.Since the deleted key will be marked specially, hence we can accordingly design our search to leave out the deleted cells.
